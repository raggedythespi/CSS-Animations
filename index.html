CSS Transitions Property

CSS transitions allows you to change property values smoothly, over a given duration.

Following are the properties used for transitions:
         (a)Transition 
         (b)Transition-delay
         (c)Transition-duration
         (d)Transition-property 
         (e)Transition-timing-function    

         To create a transition effect, you must specify two things:
         (1)the CSS property you want to add an effect to
         (2)the duration of the effect

         Note:If it is not specified, the transition will have no effect, because the default value is 0.

For example: if width & height equals to 100px and the transition effect is specified (let's say width is set to 2 seconds). The transition will only start if the specified CSS proeprty (width) changes value for the hover. See below.

Specify the speed curve of the transition

The transition-timing-function property specifies the speed curve of the transition effect. 

This timing function property has following values:
         (a)ease: specifies a transition effect with a slow start, then fast, then end slowly (this is default)
         (b)linear: specifies same speed from start to end
         (c)ease-in: slow start
         (d)ease-out: slow end
         (e)ease-in-out: slow start & end
         (d)cubic bezier (n,n,n,n)let's you define your own value

CSS Animations

CSS allows animation of HTML elements wuthiout using JavaScript

An element lets an element gradually change from one style to another.

To use CSS animation, you must first specify some keyframes for the animation. Keyframes hold what style an element will have at certain points.

THe @keyframes Rule

Once you specify CSS styles inside the @keyframes rule, the animation will gradually change from one style to another style at certain times.

To get an animation to work, you must bind the animation to an element.

The animation-duration property defines how long an animation should take time to complete. If the animation-duration property is not specified, no animation will occur, because the default value is 0seconds.

It is also possible to use percent rather than using from and to which only specifies 2 kind of percentages (starts at 0% and end at 100%), by using percentage, you can add as many styles as you want.

Specify the fill-mode For an Animation

Since CSS animation does not affect an element before the first keyframe is played or after the last keyframe is played. The animation-fill-mode property can override this behaviour.

The animation-fill-mode property specifies a style for the target element when the animation is not playing (before it starts, after it ends, or both.) 

The animation-fill-mode property can have the following values:
         (a)none - default. animation will not apply any styles to the element before or after its executing.
         (b)forwards - element will retain the style values that is set by the last keyframe (depends on animation-direction and animation-iteration-count.)
         (c)backwards - element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during the animation-delay period.
         (d)both - the animation will follow the rules for both forwards and backwards, extending the animation properties in both directions.
